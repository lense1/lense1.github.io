프로젝트를 통해 배운 것

4개의 프로젝트를 거치며 쌓인 기술적 경험과 협업 인사이트입니다.

Java · Python 이중 백엔드

VX에서 Spring Boot로 인증·API 설계를 담당하고, Stock+er에서 FastAPI로 결제·비즈니스 로직을 구현하며 두 생태계를 모두 실전에서 경험했습니다. Java의 타입 안정성과 Python의 생산성을 상황에 맞게 선택할 수 있는 시각을 갖게 됐습니다.

AI 서버 연동 설계

VX에서 Spring Boot ↔ FastAPI AI 서버 간 HTTP 통신 구조를 설계하며 마이크로서비스 아키텍처를 실제로 경험했습니다. AI 모델의 결과값을 비즈니스 로직과 연결하는 방식, 서비스 레이어 분리의 중요성을 배웠습니다.

결제 시스템 구현

Stock+er에서 Toss Payments 결제 승인·취소 API를 직접 연동하며 금전이 오가는 시스템의 신뢰성 설계를 고민했습니다. 결제 상태를 READY → APPROVED → CANCELED 단계로 명확히 관리하는 중요성을 체감했습니다.

5~7인 팀 협업

VX(5인)와 Stock+er(7인) 모두에서 백엔드 핵심 역할을 담당하며 팀원 간 API 명세 공유와 역할 분담의 중요성을 배웠습니다. 프론트와 백엔드 경계를 명확히 정의하는 것이 개발 속도에 얼마나 큰 영향을 미치는지 직접 경험했습니다.
